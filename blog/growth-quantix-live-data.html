Growth Quantix — Wiring Live Data to Modular Trading Signals

FastAPI WebSockets Redis Trading

In high-velocity algorithmic trading, ingesting market data is only half the battle. The true architectural challenge lies in transforming raw, noisy tick streams into actionable trading signals with sub-millisecond execution latency.

This case study breaks down how I engineered the signal generation engine behind Growth Quantix using Python FastAPI, Redis Pub/Sub, and modular strategy runners.

The Architectural Challenge

When processing live market feeds at 10,000+ ticks per second, tightly coupling your data ingestion layer with your strategy computation layer leads to disaster. If an RSI or MACD calculation blocks the thread for even 10 milliseconds, incoming WebSocket packets buffer, leading to stale price execution or connection drops.

Decoupling via Redis Pub/Sub Bus

To solve thread contention, I architected a completely asynchronous, event-driven data bus. Ingestion workers do nothing but decode binary packets and publish standardized JSON events to Redis channels.

Modular Strategy Engine

Each trading strategy runs as an independent consumer worker listening to relevant symbol channels. Strategies inherit from a base SignalStrategy class, ensuring clean separation of mathematical modeling from order execution. 

import json, asyncio
from redis.asyncio import Redis

class MomentumStrategy:
    def __init__(self, symbol: str, threshold: float):
        self.symbol = symbol
        self.threshold = threshold
        self.prices = []

    async def run(self, redis: Redis):
        pubsub = redis.pubsub()
        await pubsub.subscribe(f"ticks:{self.symbol}")
        
        async for message in pubsub.listen():
            if message["type"] == "message":
                tick = json.loads(message["data"])
                await self.evaluate(tick["price"])

    async def evaluate(self, price: float):
        self.prices.append(price)
        if len(self.prices) > 20:
            self.prices.pop(0)
            # Calculate simple moving average momentum
            sma = sum(self.prices) / len(self.prices)
            if price > sma * (1 + self.threshold):
                print(f"[SIGNAL] BUY {self.symbol} @ {price}")

Managing Concurrency with FastAPI

FastAPI serves as the orchestration layer and REST control plane. Through FastAPI's dependency injection and background task management, system administrators can dynamically start, pause, or adjust strategy parameters without dropping the underlying market feed connections.

Performance Outcomes

  • Zero Tick Loss: Decoupling ingestion from computation eliminated WebSocket buffer overflows.
  • Linear Scalability: New strategies can be spun up as separate container processes without impacting existing runners.
  • Sub-5ms End-to-End Latency: From tick arrival at the ingestion worker to signal emission at the execution bus.
