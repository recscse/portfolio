S3 + CloudFront: Production-Grade Static Hosting for Pennies

AWS S3 CloudFront DevOps

Hosting modern frontend applications or developer portfolios doesn't require expensive virtual private servers. Combining Amazon S3 for storage with AWS CloudFront for global CDN distribution delivers enterprise-grade performance, SSL encryption, and infinite scalability for mere cents per month.

Here are my architectural setup notes and best practices for configuring a hardened static delivery pipeline.

Delivery Architecture

Never expose an S3 bucket directly to the public internet. Instead, configure the bucket as private and grant access strictly to CloudFront using an Origin Access Control (OAC).

AWS Static Hosting Topology
  ┌──────────────┐      ┌─────────────────────────┐      ┌──────────────────────┐
  │ User Browser │─────▶│ AWS CloudFront (CDN)    │─────▶│ Private S3 Bucket    │
  │ (HTTPS / SSL)│ ◀────│ (Edge Caching + WAF)    │ ◀────│ (Origin Access Control)│
  └──────────────┘      └─────────────────────────┘      └──────────────────────┘

Hardening S3 Bucket Policies

With Origin Access Control (OAC), your S3 bucket policy blocks all public IAM requests while explicitly permitting CloudFront to retrieve objects on behalf of users. 

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "AllowCloudFrontServicePrincipal",
            "Effect": "Allow",
            "Principal": {
                "Service": "cloudfront.amazonaws.com"
            },
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::my-portfolio-bucket/*",
            "Condition": {
                "StringEquals": {
                    "AWS:SourceArn": "arn:aws:cloudfront::123456789012:distribution/EDFDVBD632BHDS5"
                }
            }
        }
    ]
}

Optimizing Edge Caching & Cache-Control

To achieve blazing fast load times, configure HTTP headers during deployment. For static assets with hashed filenames (like compiled CSS/JS), set long caching ttl. For HTML files, force frequent edge revalidation.

  • HTML files: Cache-Control: max-age=3600, must-revalidate
  • Assets (Images/CSS/JS): Cache-Control: max-age=31536000, immutable

Automated CI/CD Cache Invalidation

When deploying updates via GitHub Actions or Jenkins, always trigger an automated CloudFront invalidation for changed HTML paths to guarantee visitors immediately see the latest release. 

# Deploy synced assets to S3
aws s3 sync ./dist s3://my-portfolio-bucket --delete

# Invalidate HTML index cache across edge nodes
aws cloudfront create-invalidation --distribution-id EDFDVBD632BHDS5 --paths "/*.html" "/index.html"

Conclusion

This serverless hosting topology combines sub-50ms global latencies with zero ongoing server maintenance. It represents the gold standard for hosting static assets and single-page applications.
