QR-based Document Handoffs: Building 100% Audit Trails in Spring Boot

Security PostgreSQL Spring Boot

Bridging physical workflows with digital security requires careful attention to authentication boundaries. When transferring confidential legal or corporate files via physical QR code handoffs, ensuring complete non-repudiation and access auditability is paramount.

This post details the security architecture behind my Document Sharing System, illustrating how ephemeral JWT tokens and immutable PostgreSQL ledger entries prevent unauthorized data leaks.

Threat Modeling Physical QR Handoffs

Static QR codes printing URL links are inherently insecure. If a QR code is photographed or intercepted by a bad actor, they gain perpetual access to sensitive document buckets. To counter this, QR codes generated by our backend encode single-use, time-bound cryptographic nonces rather than direct file URLs.

Token Exchange & Verification Protocol

When a recipient scans a QR code, their mobile client initiates a cryptographic handshake with our Spring Boot API gateway.

Immutable Audit Trails in PostgreSQL

Every verification attempt—whether successful or rejected—is recorded in an append-only PostgreSQL ledger. By indexing scanner IP addresses, user-agent headers, timestamp offsets, and GPS geofence markers (when granted), security auditors can reconstruct exact chains of custody. 

CREATE TABLE document_audit_ledger (
    audit_id BIGSERIAL PRIMARY KEY,
    document_id UUID NOT NULL,
    nonce_token VARCHAR(64) UNIQUE NOT NULL,
    requester_ip VARCHAR(45) NOT NULL,
    device_fingerprint VARCHAR(255),
    access_status VARCHAR(20) NOT NULL, -- 'GRANTED', 'EXPIRED', 'DENIED'
    accessed_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_doc_audit ON document_audit_ledger(document_id, accessed_at DESC);

Role-Based Access Control (RBAC) Integration

Beyond nonce verification, Spring Security filters enforce strict RBAC boundaries. Even if a valid QR token is scanned, the underlying document download is aborted unless the scanning authenticated session possesses the required organizational security clearance tag.

Summary

Combining single-use QR nonces with ephemeral CloudFront signed URLs and immutable PostgreSQL audit logs creates an airtight, enterprise-ready handoff mechanism that satisfies rigorous compliance frameworks.
